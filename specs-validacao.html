Sistema de Validação de Grupos de Pesquisa

Módulo derivado do Dashboard PRPGI que processa dados de produção acadêmica, aplica critérios institucionais de validação conforme o Regulamento Geral dos Grupos de Pesquisa do IFBA (aprovado pelo CONSEPE) e diretrizes do CNPq/DGP, e gera relatórios de certificação.

1. Base Legal

DocumentoDescrição
Regulamento Geral dos Grupos de Pesquisa do IFBA Aprovado pelo CONSEPE/IFBA. Define criação, certificação, avaliação bienal, pontuação mínima e tabela de produtividade (Anexo I). Fonte: regulamento_dos_grupos_de_pesquisa1.pdf
Lei nº 11.892/2008 Institui a Rede Federal de Educação Profissional, Científica e Tecnológica
Diretrizes CNPq — DGP Diretório dos Grupos de Pesquisa: critérios de atipicidade, certificação e atualização

3.3 Critérios IFBA — Pontuação Mínima (Scoring)

A pontuação é calculada conforme o Anexo I — Tabela de Pontuação da Produtividade. Cada item de produção com autoria de mais de um membro do grupo só é pontuado uma vez (Art. 16, III).

IDCritérioRegraPesoRef.
IFBA-08 Pontuação mínima: 1–2 anos TotalPontos / NumMembros >= 6 (grupos com 1 a <2 anos) Obrigatório Art. 16, IV-a
IFBA-09 Pontuação mínima: 2–4 anos TotalPontos / NumMembros >= 12 (grupos com 2 a 4 anos) Obrigatório Art. 16, IV-b
IFBA-10 Pontuação mínima: >4 anos TotalPontos / NumMembros >= 20 (grupos com mais de 4 anos) Obrigatório Art. 16, IV-c

3.6 API do Motor

class ValidadorGrupo {
    constructor(grupo, dados, periodoSelecionado)

    // Executa todos os critérios e retorna resultado
    validar() → ResultadoValidacao

    // Calcula pontuação total conforme Anexo I
    calcularPontuacao() → {
        totalPontos: number,
        numMembros: number,
        pontosPorMembro: number,
        pontuacaoMinimaRequerida: number,  // 6, 12 ou 20
        atingiuMinimo: boolean,
        detalhamento: {
            projetos: number,
            bibliografica: number,
            tecnica: number,
            cultural: number,
            inovacao: number,
            eventos: number,
            orientacoesConcluidas: number,
            orientacoesAndamento: number,
            bolsas: number,
            bancas: number,
            captacao: number,
            premios: number
        }
    }

    // Verifica produção científica do líder
    validarProducaoLider() → {
        eventosUltimos2Anos: number,
        publicacoesUltimos4Anos: number,
        passou: boolean
    }

    // Retorna produções cruzadas por tipo
    getProducaoCruzada() → {
        bibliografica: [...],
        tecnica: [...],
        inovacao: [...],
        concluidas: [...],
        andamento: [...],
        eventos: [...],
        bancas: [...],
        projetos: [...]
    }

    // Retorna métricas consolidadas
    getMetricas() → {
        totalProducoes: number,
        porAno: { [ano]: number },
        porTipo: { bibliografica, tecnica, inovacao, orientacoes, eventos, bancas },
        pesquisadores: number,
        estudantes: number,
        apoioTecnico: number,
        linhasPesquisa: number,
        anoCriacao: number,
        tempoFormacaoAnos: number
    }
}

interface ResultadoValidacao {
    grupo: {
        nome: string,
        area: string,
        unidade: string,
        situacao: string,
        anoCriacao: number
    }
    criterios: [
        {
            id: string,
            nome: string,
            fonte: "CNPq" | "IFBA",
            tipo: "obrigatorio" | "advertencia",
            passou: boolean,
            detalhe: string,
            artigoRef: string  // ex: "Art. 16, IV-b"
        }
    ]
    pontuacao: {
        total: number,
        membros: number,
        porMembro: number,
        minimoRequerido: number,
        atingiu: boolean
    }
    resumo: {
        totalCriterios: number,
        obrigatoriosPassados: number,
        obrigatoriosFalhos: number,
        advertencias: number,
        parecer: "CERTIFICADO" | "NAO_CERTIFICADO" | "INADIMPLENTE"
    }
    periodo: { inicio: number, fim: number }
    geradoEm: string  // ISO 8601
}

3.7 Lógica de Parecer

ParecerCondiçãoConsequência (Regulamento)
CERTIFICADO Todos os critérios obrigatórios passaram + pontuação mínima atingida Manutenção da certificação institucional
CERTIFICADO c/ Advertências Todos obrigatórios passaram + pontuação atingida, mas há ≥ 1 advertência Manutenção da certificação, mas com recomendações
NÃO CERTIFICADO ≥ 1 critério obrigatório falhou OU pontuação mínima não atingida Prazo de 2 anos para adequação com plano de ação (Art. 17). Se persistir, grupo deve ser excluído do DGP/CNPq
INADIMPLENTE Relatório bienal não entregue no prazo Bloqueio de todos os membros em programas PRPGI + perda de certificação. Efeitos cessam 3 meses após entrega (Art. 15, §4-5)

5.2 Geração de PDF

Usar window.print() com CSS @media print para gerar PDF nativo do navegador. Sem dependência de biblioteca externa. O template de relatório terá estilos específicos para impressão: margens adequadas, cores otimizadas, quebras de página controladas.

6. Regras de Negócio

6.1 Cruzamento de Dados

Os grupos DGP possuem campo Unidade (ex: "IFBA - Salvador"). O cruzamento com produções usa o mapeamento Unidade → Campus Code → dados de produção, reutilizando a lógica CAMPUS_TO_CITY existente no dashboard.

6.2 Período de Avaliação

O período selecionado pelo usuário filtra todas as produções para cálculo de pontuação. Os critérios temporais específicos usam janelas fixas:

  • Eventos do líder: últimos 2 anos (Art. 4, §7-a)
  • Publicações do líder: últimos 4 anos (Art. 4, §7-b)
  • Pontuação do grupo: período bienal de avaliação definido pela PRPGI (Art. 16)

6.3 Cálculo de Tempo de Formação

TempoFormacao = AnoAtual - AnoCriacao

Tipo de grupoFaixaPontuação mínima/membro
Novo1 ≤ tempo < 2 anos6 pontos
Em consolidação2 ≤ tempo ≤ 4 anos12 pontos
Consolidadotempo > 4 anos20 pontos

6.4 Pontuação por Autoria Múltipla

Cada item de produção que tenha autoria de mais de um membro do grupo só poderá ser pontuado uma vez e em apenas um item do Anexo I (Art. 16, III). O sistema deve detectar duplicatas por título/DOI e deduzir automaticamente.

6.5 Múltiplos Campi

Se um grupo tiver pesquisadores vinculados a múltiplos campi, as produções de todos os campi relevantes são agregadas. O campo Unidade do grupo define o campus primário para fins de mapeamento.

6.6 Grupos com menos de 1 ano

Grupos com menos de 1 ano de formação devem apresentar relatórios, mas não são pontuados para efeito de critério de certificação (Art. 15, §6).

6.7 Recuperação de Certificação

Grupos "Não Certificado" têm prazo de até 2 anos para adequação, devendo apresentar plano de ação. Se atenderem aos requisitos, retornam ao status "Certificado". Se permanecerem "Não Certificado" por mais de 2 anos, o líder deve excluir o grupo do DGP/CNPq (Art. 17).

7. Rastreabilidade

Cada relatório deve incluir no rodapé:

  • Versão dos dados: meta.generatedAt
  • Fontes utilizadas: meta.sourceDates (label + data de modificação)
  • Versão do motor de validação (semver)
  • Referência ao regulamento: "Regulamento Geral dos Grupos de Pesquisa do IFBA — aprovado pelo CONSEPE"

8. Questões em Aberto

  • Autenticação: O sistema precisa de login ou é aberto para consulta?
  • Histórico: Deve haver armazenamento de relatórios gerados anteriormente?
  • Deduplicação: Como identificar produções duplicadas entre membros do mesmo grupo (mesmo título, DOI diferente)?
  • Qualis desatualizado: Como tratar periódicos sem Qualis ou com Qualis não atualizado (item 2.8)?
  • Produções culturais/artísticas: Os dados do Lattes/DGP capturam adequadamente esses tipos de produção?
